## 1. 引言

### 1.1 编写目的

本需求说明书旨在明确碳排放管理系统的功能需求、性能要求、界面设计规范等内容，为系统开发、测试、验收提供依据。本文档适用于：

- 产品经理：了解产品功能规划
- 开发团队：明确开发任务和技术要求
- 测试团队：制定测试计划和测试用例
- 项目管理人员：进行项目规划和进度控制
- 客户和决策者：了解系统功能和价值

### 1.2 项目背景

随着全球气候变化问题日益严峻，各国政府纷纷制定碳中和目标。中国承诺在2030年前实现碳达峰，2060年前实现碳中和。企业作为碳排放的主要来源，面临着越来越严格的碳排放监管要求。

为帮助企业有效管理碳排放数据、降低碳排放量、合规完成碳配额交易，开发一套完整的碳排放管理系统势在必行。

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| 碳排放 | 企业在生产过程中产生的温室气体排放，主要以二氧化碳当量（CO₂e）计量 |
| 碳配额 | 政府分配给企业的碳排放许可额度，以吨CO₂为单位 |
| 碳交易 | 企业之间买卖碳配额的市场行为 |
| 碳达峰 | 碳排放量达到历史最高值后开始下降的转折点 |
| 碳中和 | 企业或地区通过减排和碳汇等方式，使净碳排放量为零 |
| 范围一排放 | 企业直接产生的温室气体排放 |
| 范围二排放 | 企业外购电力、热力产生的间接排放 |
| 范围三排放 | 企业价值链上下游产生的其他间接排放 |
| CDM | 清洁发展机制，联合国碳交易机制之一 |
| CCER | 中国核证自愿减排量 |

### 1.4 参考资料

- 《温室气体核算体系》（GHG Protocol）
- 《企业温室气体排放核算方法与报告指南》
- 《碳排放权交易管理办法》

## 2. 产品概述

### 2.1 产品定位

碳排放管理系统是一款面向企业环保部门、工厂管理者、政府监管机构的专业级SaaS平台，提供碳排放数据采集、监测、分析、报告、交易等全流程管理功能。

### 2.2 产品目标

#### 2.2.1 业务目标

1. **合规管理**：帮助企业满足政府碳排放监管要求，避免违规处罚
2. **数据透明**：实现碳排放数据的实时采集、监测和可视化展示
3. **减排优化**：通过数据分析提供减排建议，降低企业碳排放量
4. **成本控制**：优化碳配额使用，降低碳交易成本
5. **决策支持**：为管理层提供数据支持，辅助战略决策

#### 2.2.2 用户目标

1. **环保部门**：高效管理碳排放数据，快速生成合规报告
2. **工厂管理者**：实时掌握排放情况，及时响应预警
3. **政府监管机构**：获取准确的企业排放数据，加强监管

### 2.3 产品范围

#### 2.3.1 包含范围

- 碳排放数据采集与录入
- 实时监测与预警
- 数据分析与可视化
- 自动化报告生成
- 碳配额管理
- 碳交易辅助
- 减排方案管理
- 企业信息管理
- 用户权限管理

#### 2.3.2 不包含范围

- 碳排放权交易平台（仅提供交易辅助功能）
- 碳金融衍生品交易
- 第三方碳核查服务
- 碳足迹认证服务

### 2.4 产品价值

| 利益相关方 | 价值主张 |
|------------|----------|
| 企业 | 降低合规风险、优化减排成本、提升品牌形象 |
| 政府 | 提高监管效率、获取准确数据、推动政策落地 |
| 社会 | 促进碳减排、应对气候变化、实现可持续发展 |

## 3. 用户角色分析

### 3.1 用户画像

#### 3.1.1 企业环保部门主管

**基本信息**
- 年龄：35-50岁
- 职位：环保部门经理/主管
- 教育背景：环境工程、化学工程等相关专业本科及以上学历
- 工作经验：5年以上环保管理经验

**工作职责**
- 制定企业碳排放管理策略
- 监督碳排放数据采集和报告
- 协调各部门完成减排目标
- 对接政府监管部门

**痛点需求**
- 数据分散，难以统一管理
- 报告编制工作量大
- 缺乏专业的数据分析工具
- 减排方案制定缺乏数据支撑

**使用场景**
- 每日查看碳排放数据概览
- 每月审核碳排放报告
- 每季度分析排放趋势
- 年度制定减排计划

#### 3.1.2 碳排放数据录入员

**基本信息**
- 年龄：25-35岁
- 职位：环保专员/数据录入员
- 教育背景：环境科学、统计学等相关专业
- 工作经验：1-3年

**工作职责**
- 采集各排放源数据
- 录入系统并保证数据准确性
- 整理和归档相关凭证
- 配合数据审核工作

**痛点需求**
- 数据来源多，格式不统一
- 手工录入效率低
- 数据校验困难
- 历史数据查询不便

**使用场景**
- 每日录入排放数据
- 上传监测凭证
- 查询历史录入记录
- 处理数据审核反馈

#### 3.1.3 工厂生产管理者

**基本信息**
- 年龄：30-45岁
- 职位：生产车间主任/厂长
- 教育背景：机械工程、工业工程等专业
- 工作经验：5年以上生产管理经验

**工作职责**
- 管理生产车间日常运营
- 监控生产设备运行状态
- 控制生产成本
- 执行减排措施

**痛点需求**
- 不了解碳排放对生产的影响
- 缺乏实时排放数据
- 预警信息不及时
- 减排措施执行困难

**使用场景**
- 实时查看车间排放数据
- 接收排放预警通知
- 查看减排措施执行进度
- 分析生产与排放关系

#### 3.1.4 政府监管人员

**基本信息**
- 年龄：30-50岁
- 职位：生态环境局工作人员
- 教育背景：环境管理、公共管理等专业
- 工作经验：3年以上

**工作职责**
- 监管企业碳排放合规情况
- 审核企业碳排放报告
- 执行碳排放配额管理
- 开展碳排放核查

**痛点需求**
- 企业数据上报不及时
- 数据真实性难以验证
- 监管效率低
- 缺乏有效的监管工具

**使用场景**
- 查看企业排放数据
- 审核企业报告
- 下达监管指令
- 分析区域排放情况

## 4. 功能需求

### 4.1 数据概览模块（Dashboard）

#### 4.1.1 功能概述

数据概览模块是系统的首页，为用户提供碳排放数据的整体视图，包括关键指标、趋势图表、待办事项等。

#### 4.1.2 功能列表

| 功能编号 | 功能名称 | 优先级 | 功能描述 |
|----------|----------|--------|----------|
| F1.1.1 | 欢迎信息展示 | P0 | 显示用户姓名、职位、所属企业 |
| F1.1.2 | 本月碳排放量 | P0 | 展示本月累计碳排放量及同比变化 |
| F1.1.3 | 今日排放量 | P0 | 展示今日实时碳排放量 |
| F1.1.4 | 目标完成率 | P0 | 展示年度减排目标完成进度 |
| F1.1.5 | 减排量统计 | P0 | 展示累计减排量 |
| F1.1.6 | 排放趋势图 | P1 | 展示近7天/30天/3个月排放趋势 |
| F1.1.7 | 排放源分布 | P1 | 展示各排放源占比情况 |
| F1.1.8 | 待办事项 | P0 | 展示待处理的任务列表 |
| F1.1.9 | 减排动态 | P1 | 展示减排项目进展情况 |
| F1.1.10 | 快捷入口 | P1 | 提供常用功能的快捷入口 |

#### 4.1.3 详细功能说明

**F1.1.2 本月碳排放量**

输入：
- 无（自动从数据库获取）

处理逻辑：
1. 系统自动汇总本月所有排放源数据
2. 计算本月累计排放量
3. 与上月同期数据对比，计算变化率
4. 判断变化趋势（上升/下降）

输出：
- 本月累计排放量（吨CO₂）
- 同比变化率（%）
- 趋势图标（上升/下降箭头）

业务规则：
- 数据实时更新，每小时刷新一次
- 变化率保留一位小数
- 下降显示绿色，上升显示红色

**F1.1.6 排放趋势图**

输入：
- 时间范围选择（近7天/30天/3个月）

处理逻辑：
1. 根据选择的时间范围查询历史数据
2. 按日期分组统计排放量
3. 生成趋势折线图
4. 标注最高值、最低值、平均值

输出：
- 趋势折线图
- 最高值、最低值、平均值数据
- 日期标签

业务规则：
- 默认显示近7天数据
- 支持用户自定义时间范围
- 数据缺失时显示提示信息

**F1.1.8 待办事项**

输入：
- 无（自动从系统获取）

处理逻辑：
1. 查询当前用户的待办任务
2. 按优先级和截止日期排序
3. 筛选出紧急任务
4. 显示任务类型、标题、截止日期

输出：
- 待办任务列表
- 任务类型图标
- 截止日期
- 紧急程度标识

业务规则：
- 最多显示5条待办事项
- 超期任务标红显示
- 点击任务跳转到对应处理页面

#### 4.1.4 界面原型

参见：`pages/dashboard.html`

#### 4.1.5 数据要求

| 数据项 | 数据类型 | 数据来源 | 更新频率 |
|--------|----------|----------|----------|
| 本月排放量 | 数值型 | 排放数据表 | 实时 |
| 今日排放量 | 数值型 | 排放数据表 | 实时 |
| 目标完成率 | 百分比 | 目标表+排放数据表 | 每日 |
| 趋势数据 | 时间序列 | 排放数据表 | 每小时 |
| 待办事项 | 列表 | 任务表 | 实时 |

#### 4.2.1 功能概述

数据录入模块提供碳排放数据的采集、录入、导入功能，支持多种排放源类型的数据录入。

#### 4.2.2 功能列表

| 功能编号 | 功能名称 | 优先级 | 功能描述 |
|----------|----------|--------|----------|
| F2.1.1 | 快速录入入口 | P0 | 提供常用排放类型的快捷入口 |
| F2.1.2 | 排放类型选择 | P0 | 选择排放类型（范围一/二/三） |

| F2.1.6 | 数据来源选择 | P1 | 选择数据来源类型 |
| F2.1.7 | 附件上传 | P1 | 上传监测凭证、发票等附件 |
| F2.1.8 | 备注说明 | P2 | 添加数据说明备注 |
| F2.1.9 | 保存草稿 | P1 | 保存未完成的数据录入 |

| F2.1.13 | 最近录入记录 | P1 | 展示最近录入的数据记录 |

#### 4.2.3 详细功能说明

**F2.1.4 排放量录入**

输入：
- 排放量数值（必填）
- 计量单位（默认：吨CO₂）

处理逻辑：
1. 校验输入是否为有效数字
2. 校验数值范围是否合理（0-999999）
3. 自动换算单位（支持吨、千克等）
4. 计算对应的碳配额消耗

输出：
- 格式化后的排放量
- 单位换算结果
- 配额消耗提示

业务规则：
- 必须为正数
- 保留两位小数
- 超出合理范围时提示确认

**F2.1.11 批量导入**

输入：
- Excel文件（.xlsx格式）

处理逻辑：
1. 校验文件格式和大小（≤10MB）
2. 解析Excel文件内容
3. 校验数据格式和完整性
4. 显示校验结果（成功/失败条数）
5. 用户确认后批量写入数据库

输出：
- 导入结果统计
- 错误数据列表
- 成功导入的数据预览

业务规则：
- 单次导入不超过1000条
- 重复数据提示用户选择处理方式
- 导入失败的数据提供下载功能

#### 4.2.4 界面原型

参见：`pages/data-entry.html`

#### 4.2.5 数据要求

| 数据项 | 数据类型 | 是否必填 | 校验规则 |
|--------|----------|----------|----------|
| 排放类型 | 枚举型 | 是 | 范围一/二/三 |

| 备注 | 文本 | 否 | ≤500字 |

### 4.3 实时监测模块（Monitoring）

#### 4.3.1 功能概述

实时监测模块提供碳排放数据的实时监控功能，包括排放速率、监测点状态、预警信息等。

#### 4.3.2 功能列表

| 功能编号 | 功能名称 | 优先级 | 功能描述 |
|----------|----------|--------|----------|
| F3.1.1 | 当前排放速率 | P0 | 实时显示当前排放速率 |
| F3.1.2 | 温度监测 | P1 | 显示排放口温度数据 |
| F3.1.3 | 流速监测 | P1 | 显示废气流速数据 |
| F3.1.4 | CO₂浓度监测 | P0 | 显示CO₂浓度数据 |
| F3.1.5 | 设备效率监测 | P1 | 显示处理设备运行效率 |
| F3.1.6 | 监测点状态 | P0 | 显示各监测点运行状态 |
| F3.1.7 | 预警信息 | P0 | 显示排放预警信息 |
| F3.1.8 | 实时趋势图 | P1 | 显示排放速率实时趋势 |
| F3.1.9 | 历史数据查询 | P1 | 查询历史监测数据 |
| F3.1.10 | 数据导出 | P2 | 导出监测数据 |

#### 4.3.3 详细功能说明

**F3.1.1 当前排放速率**

输入：
- 无（自动从监测设备获取）

处理逻辑：
1. 从监测设备实时获取排放数据
2. 计算当前排放速率（吨CO₂/小时）
3. 与正常范围对比
4. 判断状态（正常/偏高/偏低）

输出：
- 当前排放速率
- 正常范围参考值
- 状态标识（正常/注意/警告）

业务规则：
- 数据每5秒更新一次
- 超出正常范围时自动预警
- 数据异常时显示提示信息

**F3.1.7 预警信息**

输入：
- 无（自动从系统获取）

处理逻辑：
1. 实时监测各指标数据
